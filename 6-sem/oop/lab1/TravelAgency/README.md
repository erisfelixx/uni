# Travel Agency Web Application

Full-stack SPA для управління туристичною агенцією. Проєкт створено з використанням архітектури REST API, де бекенд реалізовано на чистих Java Servlets, а фронтенд — на базі React.

---

## Основний функціонал

- **Каталог турів:** Перегляд доступних путівок із поділом на категорії (Відпочинок, Екскурсія, Шопінг).
- **Керування доступом на основі ролей (RBAC):**
    - **Клієнт (CUSTOMER):** Може переглядати тури, бронювати їх та бачити власну історію замовлень.
    - **Турагент (AGENT):** Має доступ до панелі керування для створення нових турів та визначення їх як "гарячих".
- **Система лояльності:** Автоматичний розрахунок та застосування знижки 10% для постійних клієнтів (починаючи з третього бронювання).
- **Безпека:** Захист маршрутів та API-ендпоінтів за допомогою JWT-токенів.

## Технологічний стек

### Backend (Java)

| Категорія | Технологія |
|---|---|
| Мова | Java 21 |
| Архітектура | Java EE (Servlets API 6.1.0, Filters), MVC, RESTful API |
| База даних | PostgreSQL, JDBC (без ORM), патерн DAO |
| Безпека | JWT (Auth0) |
| Інструменти | Maven, Lombok, MapStruct |
| Логування | Log4j2 |
| Тестування | JUnit 5, Mockito |
| Сервер | Apache Tomcat |

### Frontend (React)

| Категорія | Технологія |
|---|---|
| Бібліотека | React 18 |
| Збірка | Vite |
| Маршрутизація | React Router DOM |
| HTTP-клієнт | Axios (з перехоплювачами для JWT) |

## Структура проєкту

Проєкт використовує підхід Monorepo:

```
/src/main/java/ua/oop/...       — вихідний код Java-бекенду
/src/main/resources             — SQL-скрипти та конфігурація логування
/src/test/java/ua/oop/...       — модульні тести для сервісів та контролерів
/frontend                       — вихідний код React-додатка
```

## Інструкція із запуску

### 1. Налаштування бази даних

1. Встановіть та запустіть PostgreSQL.
2. Створіть нову базу даних для проєкту.
3. Виконайте SQL-скрипт з файлу `src/main/resources/init.sql` для створення таблиць та наповнення початковими даними.
4. Оновіть облікові дані для підключення у класі `DatabaseConnection.java`.

### 2. Запуск Backend-сервера

1. Відкрийте проєкт у вашому IDE (наприклад, IntelliJ IDEA).
2. Завантажте залежності Maven.
3. Налаштуйте Run Configuration для локального сервера Apache Tomcat.
4. Запустіть сервер. API буде доступне за адресою:

```
http://localhost:8080/TravelAgency_war_exploded/api/...
```

### 3. Запуск Frontend-клієнта

```bash
cd frontend
npm install
npm run dev
```

Відкрийте [http://localhost:5173](http://localhost:5173) у браузері.
